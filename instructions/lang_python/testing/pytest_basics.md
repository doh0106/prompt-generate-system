## [Python] Pytest 기본 테스트 코드 작성 규칙

Python의 `pytest` 프레임워크를 사용하여 테스트 코드를 작성할 때, 아래의 규칙들을 반드시 준수하여 명확하고 신뢰성 높은 테스트를 생성해야 한다.

### 1. 네이밍 컨벤션 (Naming Convention)
- 테스트 파일의 이름은 `test_*.py` 또는 `*_test.py` 형식으로 작성하라.
- 테스트 함수의 이름은 `test_`로 시작해야 한다.
- 함수의 이름은 테스트의 의도를 명확히 설명하도록 서술형으로 작성하라. (예: `test_user_model_creation_with_invalid_email_should_raise_value_error`)

### 2. AAA 패턴 준수 (Arrange-Act-Assert)
- 모든 테스트 함수는 아래의 세 부분으로 명확히 구조화되어야 한다.
  - **Arrange (준비)**: 테스트에 필요한 변수, 객체, 상태 등을 설정하는 부분.
  - **Act (실행)**: 테스트하려는 실제 함수나 메서드를 호출하는 부분.
  - **Assert (검증)**: 실행(Act)의 결과가 예상과 일치하는지 확인하는 부분.

### 3. 다양한 케이스 테스트 (Test Case Diversity)
- **정상 케이스 (Happy Path)**: 의도된 대로 동작하는 가장 일반적인 경우를 테스트하라.
- **엣지 케이스 (Edge Cases)**: 경계값(0, -1, 빈 리스트, None 등)을 사용하여 예외적인 상황을 테스트하라.
- **오류 케이스 (Error Cases)**: 특정 조건에서 의도한 대로 예외(Exception)가 발생하는지 `pytest.raises`를 사용하여 반드시 테스트하라.

### 4. 테스트의 독립성 (Independence)
- 각 테스트 함수는 다른 테스트의 결과나 상태에 의존해서는 안 된다.
- 모든 테스트는 독립적으로, 어떤 순서로 실행되더라도 항상 동일한 결과를 내야 한다.
